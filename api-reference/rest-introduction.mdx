https://tapi.lightcone.xyz/api

## Protocol

Lightcone's CLOB is a hybrid-decentralized orderbook. Operators match orders offchain, while asset issuance, settlement and signature verification are executed onchain, non-custodially and trustlessly. Orders are Ed25519-signed payloads.

Each market specifies **_Deposit Tokens_** available for trading _(e.g. USDC, wBTC)_. Deposit Tokens are held as collateral, against which **_Outcome Tokens_** are minted — one per possible outcome _(e.g. USDC => [<Badge size ="sm" className="badge-r">**A-USDC**</Badge>, <Badge size ="sm" className="badge-d">**B-USDC**</Badge>])_ — via a **_split_**. A full set of Outcome Tokens can **_merge_** back into the underlying Deposit Token.

Orderbooks facilitate swaps between **_Outcome Token pairs_** of the same outcome universe _(e.g. <Badge size ="sm" className="badge-r">**A-USDC**</Badge>/<Badge size ="sm" className="badge-r">**A-BTC**</Badge>)_.

After a market **_settles_**, the winning Outcome Tokens become **_redeemable 1:1_** for their Deposit Tokens, while all other Outcome Tokens become worthless.

## Deposits & withdrawals

The Lightcone program supports two paths to deposit and withdraw collateral.

**Global deposit pool** — Deposit once, create limit orders across any market with the same capital.  Collateral is only split into conditional tokens when an order fills, and unfillable resting orders are automatically cancelled. Recommended for market makers and set as the default `deposit_source` on the SDK

**Market position** — Locks collateral into a specific market and mints conditional tokens at deposit time. Used by the frontend.

## Rate Limits

Per-IP rate limiting. Burst of up to 60 requests, refilled at 10 per second. These rates will change at production.
