## Order Types

### Limit Order

Every order on Lightcone is a limit order. You specify `amount_in` (what you give) and `amount_out` (what you receive), the ratio of which defines the price.

### Market Order
A market order on Lightcone is an IOC limit order with `price` set to the `price` of the worst resting order it will match against. The matching engine fills against any order at or better than this price, which provides protection against stale or cancelled orders. The frontend and sdk additionally enforces a default 10% impact protection, which can be adjusted.

### Stop and Take Profit Order

Stop-loss and take-profit are pre-signed dormant limit orders triggered by a price condition. When the last trade price crosses the trigger, the order is submitted to the book as a limit order. The UI defaults the limit price to 10% from the current price, but this can be adjusted.

## Authentication
Most of the Lightcone API is read-only and doesn't require authentication. Submitting and cancelling orders requires an Ed25519 signature. Fetching a user's senstive data requires JWT authentication.

### Signature authentication

Required for:

- <Badge color="blue">Post</Badge> orders/submit
- <Badge color="blue">Post</Badge> orders/cancel
- <Badge color="blue">Post</Badge> orders/cancel-all

Orders are cryptographically bound to the ownership of a Solana wallet. To submit or cancel orders, you must include an  signed by the relevant keypair.

1. **Create the payload** with all required fields
2. **Sign the payload** using Ed25519 with your wallet's keypair
3. **Include the signature** in the request as a hex-encoded string

#### Create Order

```rust Rust
use lightcone_sdk::program::{FullOrder, BidOrderParams};
use lightcone_sdk::api::{LightconeApiClient, SubmitOrderRequest};
use solana_sdk::signer::{keypair::Keypair, Signer};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let keypair = Keypair::new();
    let client = LightconeApiClient::new("https://tapi.lightcone.xyz");

    let mut order = FullOrder::new_bid(BidOrderParams {
        nonce: std::time::SystemTime::now()
            .duration_since(std::time::UNIX_EPOCH)?
            .as_millis() as u64,
        maker: keypair.pubkey(),
        market: market_pda,
        base_mint: base_token,
        quote_mint: quote_token,
        maker_amount: 650_000,
        taker_amount: 1_000_000,
        expiration: 0,
    });

    order.sign(&keypair);

    let response = client.submit_order(SubmitOrderRequest {
        maker: keypair.pubkey().to_string(),
        nonce: order.nonce,
        market_pubkey: market_pda.to_string(),
        base_token: base_token.to_string(),
        quote_token: quote_token.to_string(),
        side: 0,
        amount_in: order.maker_amount,
        amount_out: order.taker_amount,
        expiration: order.expiration,
        signature: hex::encode(order.signature),
        orderbook_id: "orderbook_id".to_string(),
    }).await?;

    Ok(())
}
```

### Manual signing

<Accordion title="(Advanced) Sign payload manually">

#### Create Order

Orders are signed with Ed25519 over the hex-encoded `keccak256` of a 169-byte payload:

```
Offset  Size  Field
──────  ────  ─────────────
0       8     nonce (u64 LE)
8       8     salt (u64 LE)
16      32    maker pubkey
48      32    market pubkey
80      32    base_token pubkey
112     32    quote_token pubkey
144     1     side (0=bid, 1=ask)
145     8     amount_in (u64 LE)
153     8     amount_out (u64 LE)
161     8     expiration (i64 LE)
───────────────────────────
        169   Total bytes
```

The `salt` is a unique value per order (e.g. nanosecond timestamp) ensuring distinct hashes even when all other fields match.

<CodeGroup>

```typescript TypeScript
import { Keypair } from '@solana/web3.js';
import nacl from 'tweetnacl';
import { keccak_256 } from '@noble/hashes/sha3';

const keypair = Keypair.generate();

// Build 169-byte message (see Order Hash Structure)
const buffer = new ArrayBuffer(169);
const view = new DataView(buffer);
const bytes = new Uint8Array(buffer);

view.setBigUint64(0, BigInt(nonce), true);
view.setBigUint64(8, BigInt(salt), true);
bytes.set(keypair.publicKey.toBytes(), 16);
bytes.set(market.toBytes(), 48);
bytes.set(baseToken.toBytes(), 80);
bytes.set(quoteToken.toBytes(), 112);
view.setUint8(144, side);
view.setBigUint64(145, BigInt(amountIn), true);
view.setBigUint64(153, BigInt(amountOut), true);
view.setBigInt64(161, BigInt(expiration), true);

// Hash and sign
const hash = keccak_256(bytes);
const hexHash = Buffer.from(hash).toString('hex');
const signature = nacl.sign.detached(Buffer.from(hexHash), keypair.secretKey);

// Submit
const response = await fetch('https://tapi.lightcone.xyz/api/orders/submit', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    maker: keypair.publicKey.toBase58(),
    nonce, salt, market_pubkey: market.toBase58(),
    base_token: baseToken.toBase58(), quote_token: quoteToken.toBase58(),
    side, amount_in: amountIn, amount_out: amountOut, expiration,
    signature: Buffer.from(signature).toString('hex'),
    orderbook_id: orderbookId,
  })
});
```

```python Python
from solders.keypair import Keypair
import struct, requests
from Crypto.Hash import keccak

keypair = Keypair()

# Build 169-byte message (see Order Hash Structure)
message = struct.pack('<Q', nonce)
message += struct.pack('<Q', salt)
message += bytes(maker)
message += bytes(market)
message += bytes(base_token)
message += bytes(quote_token)
message += struct.pack('B', side)
message += struct.pack('<Q', amount_in)
message += struct.pack('<Q', amount_out)
message += struct.pack('<q', expiration)

# Hash and sign
k = keccak.new(digest_bits=256)
k.update(message)
hex_hash = k.hexdigest()
signature = keypair.sign_message(hex_hash.encode('utf-8'))

# Submit
response = requests.post(
    "https://tapi.lightcone.xyz/api/orders/submit",
    json={
        "maker": str(keypair.pubkey()),
        "nonce": nonce, "salt": salt,
        "market_pubkey": str(market),
        "base_token": str(base_token), "quote_token": str(quote_token),
        "side": side, "amount_in": amount_in, "amount_out": amount_out,
        "expiration": expiration,
        "signature": bytes(signature).hex(),
        "orderbook_id": orderbook_id,
    }
)
```

### JWT authentication

Required for:

- <Badge color="green">Get</Badge> users/orders

`POST /auth/login_or_register_with_message` verifies a signed message from a Solana wallet and returns a JWT session cookie (valid for 14 days).

<Accordion title="(Recommended) Authenticate with SDK">

<CodeGroup>

```rust Rust
use lightcone_sdk::api::LightconeApiClient;
use solana_sdk::signer::keypair::Keypair;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let keypair = Keypair::new();
    let mut client = LightconeApiClient::new("https://tapi.lightcone.xyz");
    client.authenticate(&keypair).await?;
    let orders = client.get_user_orders().await?;
    Ok(())
}
```

```typescript TypeScript
import { LightconeClient } from '@lightcone/sdk';
import { Keypair } from '@solana/web3.js';

const keypair = Keypair.generate();
const client = new LightconeClient({ apiUrl: 'https://tapi.lightcone.xyz' });
const { token, userId } = await client.authenticate(keypair);
const orders = await client.getUserOrders();
```

<Accordion title="(Advanced) Authenticate manually">

Sign a message and call the login endpoint. The request body includes `message`, `signature_bs58`, and `pubkey_bytes`.

<CodeGroup>

```typescript TypeScript
import nacl from 'tweetnacl';
import bs58 from 'bs58';

const keypair = Keypair.generate();
const { nonce } = await (await fetch('https://tapi.lightcone.xyz/api/auth/nonce')).json();
const message = `Sign in to Lightcone\nNonce: ${nonce}`;
const signature = nacl.sign.detached(new TextEncoder().encode(message), keypair.secretKey);

const response = await fetch('https://tapi.lightcone.xyz/api/auth/login_or_register_with_message', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    message,
    signature_bs58: bs58.encode(signature),
    pubkey_bytes: Array.from(keypair.publicKey.toBytes()),
  }),
});

const token = response.headers.get('set-cookie')?.match(/auth_token=([^;]+)/)?.[1];
```

```python Python
from solders.keypair import Keypair
import base58, requests, re

keypair = Keypair()
nonce = requests.get('https://tapi.lightcone.xyz/api/auth/nonce').json()['nonce']
message = f"Sign in to Lightcone\nNonce: {nonce}"
signature = keypair.sign_message(message.encode('utf-8'))

response = requests.post(
    'https://tapi.lightcone.xyz/api/auth/login_or_register_with_message',
    json={
        'message': message,
        'signature_bs58': base58.b58encode(bytes(signature)).decode('utf-8'),
        'pubkey_bytes': list(bytes(keypair.pubkey())),
    }
)

token = re.search(r'auth_token=([^;]+)', response.headers.get('set-cookie', '')).group(1)
```
