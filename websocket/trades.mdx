## Overview

The `trades` channel provides live trade execution data for specified orderbooks. Each trade is broadcast immediately after execution.

All prices and sizes are returned as **decimal strings** (e.g., `"0.65"`, `"1.5"`) matching the REST API format.

## Subscribe

```json
{
  "method": "subscribe",
  "params": {
    "type": "trades",
    "orderbook_ids": ["5BNoPgYe_CrhujZLf"]
  }
}
```

### Parameters

<ParamField path="orderbook_ids" type="string[]" required>
  Array of orderbook IDs to receive trades from
</ParamField>

## Messages

Each trade execution triggers a message:

```json
{
  "type": "trades",
  "version": 0.1,
  "data": {
    "orderbook_id": "5BNoPgYe_CrhujZLf",
    "price": "0.650000",
    "size": "1.000000",
    "side": "bid",
    "timestamp": "2026-01-09T20:43:24.123Z",
    "trade_id": "trade_1736456789000"
  }
}
```

## Response Fields

<ResponseField name="orderbook_id" type="string">
  The orderbook where the trade occurred
</ResponseField>

<ResponseField name="price" type="string">
  Trade execution price as decimal string (e.g., `"0.650000"`)
</ResponseField>

<ResponseField name="size" type="string">
  Trade size as decimal string (e.g., `"1.000000"`)
</ResponseField>

<ResponseField name="side" type="string">
  Taker's side: `"bid"` (buy) or `"ask"` (sell)
  
  - `"bid"` means a market buy order took liquidity from the ask side
  - `"ask"` means a market sell order took liquidity from the bid side
</ResponseField>

<ResponseField name="timestamp" type="string">
  ISO 8601 timestamp of trade execution
</ResponseField>

<ResponseField name="trade_id" type="string">
  Unique identifier for this trade
</ResponseField>

## Unsubscribe

```json
{
  "method": "unsubscribe",
  "params": {
    "type": "trades",
    "orderbook_ids": ["5BNoPgYe_CrhujZLf"]
  }
}
```
